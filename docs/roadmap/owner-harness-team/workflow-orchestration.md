# Workflow Orchestration

## Proposal Materialization

Track how raw product requests become OpenSpec proposals, managed branches, and
lane assignments that can be claimed by the pooled coding workflow.

## Lane Execution

Track the coder and reviewer handoff contract, validation expectations, and the
repo-local skills that keep implementation consistent across reusable worktrees.

## Finalization

Track the deterministic work that happens after machine review, including
archiving approved OpenSpec changes, updating roadmap references, and
refreshing the GitHub pull request.
