# Owner Harness Team

This roadmap tracks the harness itself: deterministic orchestration, explicit
lane handoffs, and docs that stay aligned with archived implementation work.
Topics are intentionally narrow so request-level OpenSpec changes can link back
to one operational thread without re-explaining the whole system.

## Design Notes

- Keep planner, coder, reviewer, and finalization behavior explicit in code.
- Treat docs as a durable navigation layer for future implementation and review.
- Prefer small, composable topics over one large platform backlog page.
